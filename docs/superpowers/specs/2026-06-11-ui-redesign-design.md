# 格物工坊 2.0 UI 重设计方案

## 目标

将现有软件从“功能页面集合”升级为“任务驱动的信息架构 + 专业工作台 + 统一设计系统”。最终效果优先，不以现有页面数量和实现成本为限制；但落地时仍采用可分阶段验证的方式，避免一次性重构造成不可控风险。

## 当前结构分析

项目是 Electron + React + Ant Design 的本地综合管理系统。前端入口为 `src/App.tsx`，当前由横向分组菜单切换页面，主要页面在 `src/pages/` 下。

现有模块可以归纳为：

- 教务：课程表、排课列表、课程、学生、老师、机构、学校、上课地址。
- 题库：导入、预览、编辑、审核、组卷、试题篮。
- 财务：缴费、费用统计、个人资产。
- 用户与权限：管理员、被邀请者、权限。
- 系统与数据：同步、系统参数、菜单管理、操作日志。

当前 UI 的主要问题：

- 页面入口过多，题库尤其明显，导入、预览、编辑、审核、组卷被拆得过碎。
- 样式大量散落在页面内联样式和全局 CSS 中，统一调整成本高。
- 管理页模式重复，但缺少统一模板。
- 课程表、题库、财务统计是工作台型页面，和普通表格管理页应区别设计。
- 统计页的关键价值是核账追溯，不应被图表优先级压过。

## 已确认设计方向

- 整体风格：紧凑运营台为主，吸收现代清爽层次。
- 导航骨架：左侧分组导航。
- 顶部区域：页面上下文工具条，展示标题、说明、主操作、刷新/导出/同步状态。
- 默认密度：平衡紧凑。
- 颜色语气：蓝绿专业为主，吸收墨色青蓝的深色标题和细边框。
- 管理页模板：工具栏 + 表格 + 右侧抽屉；轻量字段可表格内联编辑。
- 排课工作台：左侧课程池 + 大画布，课程池可折叠，支持大画布模式。
- 统计核账页：总览指标 + 可展开明细表，图表作为辅助。
- 题库：压缩为「题库工具 / 试题库 / 组卷」，取消独立审核中心。
- 设计系统实现：Ant Design Token 统一主题 + 自定义页面模板组件 + 分阶段迁移。

## 最终信息架构

左侧一级导航改为：

1. 今日
2. 教务
3. 题库
4. 财务
5. 资源
6. 系统与数据

### 今日

聚合高频待办，不做普通仪表盘：

- 今日课程。
- 待处理问题试题。
- 待核对费用。
- 最近导入任务与异常。
- 同步状态和最近操作提醒。

### 教务

以课程表为核心工作台：

- 课程表大画布。
- 左侧课程池，支持按老师、课程状态、地点筛选。
- 课程卡片支持拖拽、复制、调整时长、右键出勤和费用。
- 学生、老师、课程详情通过抽屉或上下文入口查看，不打断排课。

### 题库

压缩为三个入口：

- 题库工具：知识树/模型树编辑、试题导入、导入质量报告、题库统计、问题试题队列。
- 试题库：筛选、搜索、题干/解析/答案大阅读区、编辑入口、问题标记、加入试题篮。
- 组卷：从试题篮进入，完整展示试卷结构，支持题号调整、试题增删、分值、答案位置和导出设置。

取消独立审核中心。原审核能力转化为：

- 问题标记。
- 待处理问题试题队列。
- 试题库置顶强提示。
- 编辑修正、删除或取消问题标记。

历史 `draft / pending / published` 等状态不应直接粗暴删除，可在实现阶段迁移为更符合个人题库维护的状态模型，例如“可用 / 有问题 / 废弃”。

### 财务

设计为核账工作台：

- 顶部指标：应收学费、老师课时费、净收入、排课数量。
- 筛选：日期、学生、老师、课程类型、状态。
- 主路径：总览 → 学生/老师汇总 → 单节课费用明细。
- 所有金额必须能追溯到排课明细，包括日期、时间段、课程、学生、老师、时长、单位、单价和总额。

### 资源

承载基础资料维护：

- 学生。
- 老师。
- 课程。
- 机构。
- 学校。
- 上课地址。
- 缴费记录。

默认使用统一管理页模板：筛选工具栏、表格、右侧抽屉表单。状态、开关、排序等轻量字段允许表格内直接编辑。

### 系统与数据

承载低频系统功能：

- 云同步。
- 权限。
- 管理员。
- 被邀请者。
- 菜单结构。
- 系统参数。
- 操作日志。
- 备份、导入导出和同步诊断。

## 设计系统

使用 Ant Design `ConfigProvider` 和主题 token 作为基础：

- 主色：专业蓝。
- 成功/正常/收入：绿色。
- 警告/待处理：橙色。
- 风险/取消/删除：红色。
- 标题与主文本：接近墨色的深色。
- 边框：细、浅、克制。
- 圆角：中等，管理工具以 6px 左右为主。
- 默认组件密度：平衡紧凑。

新增模板组件：

- `AppShell`：左侧分组导航、顶部上下文工具条、内容区。
- `PageHeaderBar`：页面标题、说明、主操作、次级操作、状态信息。
- `DataPageLayout`：筛选区、表格、抽屉表单。
- `WorkbenchLayout`：课程表、题库等工作台型页面。
- `StatsPageLayout`：筛选条、指标条、可展开明细表。
- `QuestionWorkspace`：题库工具、试题库、组卷入口的统一壳层。

## 页面模板

### 管理页模板

适用页面：学生、老师、课程、机构、学校、上课地址、缴费、权限、日志等。

布局：

- 顶部 `PageHeaderBar`。
- 第二层筛选工具栏。
- 主体表格。
- 新增/编辑使用右侧抽屉。
- 删除、危险操作使用确认弹窗。
- 状态类字段可直接在表格中切换。

### 排课工作台模板

适用页面：课程表。

布局：

- 顶部页面工具条：日期跳转、上一周/下一周、本周、大画布、刷新课程信息。
- 左侧课程池，可折叠。
- 主体两周课程画布。
- 课程卡片强调课程名、时间、状态、地点色。
- 右键菜单保留出勤和费用、状态、删除等操作。
- 刷新课程信息允许覆盖费用明细，但必须有强提示和二次确认。

### 统计核账模板

适用页面：费用统计、个人资产等。

布局：

- 筛选条。
- 指标条。
- 汇总表。
- 可展开明细表。
- 图表折叠为辅助视图。

### 题库模板

题库工具：

- 左侧或上方放知识树/模型树维护入口。
- 导入流程使用步骤式区域或宽抽屉。
- 导入质量问题形成问题队列。
- 统计信息轻量展示，不占大空间。

试题库：

- 左侧或顶部为筛选和搜索。
- 中部题目列表。
- 右侧或主区域为大阅读区，展示题干、选项、答案、解析。
- 编辑按钮放在题目内容下方，打开宽抽屉或大弹窗。
- 问题试题置顶并强提示。

组卷：

- 独立页面。
- 展示完整试卷结构。
- 支持题号调整、试题增删、分值设置、答案位置、导出。

## 实施顺序

1. 建立设计系统基础：主题 token、全局布局尺寸、基础 CSS。
2. 重构系统壳层：左侧导航、顶部上下文工具条、页面容器。
3. 抽象管理页模板并迁移资源管理类页面。
4. 重构课程表为排课工作台。
5. 重构财务统计为核账工作台。
6. 压缩题库模块为题库工具、试题库、组卷。
7. 统一系统与数据中心。
8. 做全站视觉回归、截图检查和打包发布。

## 验收标准

- 左侧导航能覆盖所有现有功能且入口减少、层级更清楚。
- 每个页面都有统一标题区和主操作区。
- 管理页新增/编辑流程统一为抽屉。
- 课程表保留现有拖拽、复制、右键费用出勤、刷新课程信息能力。
- 费用统计所有金额都能追溯到排课明细。
- 题库不再有独立审核中心，问题试题能被置顶提示并进入待处理队列。
- 试题题干、答案、解析拥有足够阅读空间。
- 组卷页保持独立完整工作流。
- 主题色、边框、表格密度、按钮尺寸全站一致。
- 构建、类型检查、现有测试通过。
