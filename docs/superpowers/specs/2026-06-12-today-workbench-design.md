# 今日工作台重构设计

## 背景

当前左侧导航中，“今日”分组下只有“今日工作台”一个子项，需要两次点击才能进入，和入口层级不匹配。今日工作台内部也存在重复入口和无效按钮：今日课程与课程表默认定位当周的能力重复，四个“查看”按钮没有带上下文跳转，进入后仍需要用户自行查找目标内容。

本次重构目标是把今日工作台改成“高频入口 + 可执行提醒”的首页：主入口负责快速进入模块，提醒卡负责直接打开需要处理的结果集。

## 导航结构

左侧导航删除只有一个子项的“今日”分组，将“今日工作台”提升为一级入口。点击一级入口后直接进入今日工作台，不再需要展开分组。

当前页面仍以 `today` 作为路由 key，避免影响已有路由和刷新逻辑。

## 页面布局

采用已确认的 A2 方案。

页面上方显示今日工作台标题、日期和同步摘要。主体分为两块：

1. 四个主入口方块：课程表、费用统计、题库、云同步。
2. 今日提醒区：今日课程信息提示、学生费用欠缴、结课学生学费剩余、题库问题试题编辑、云同步情况。

主入口方块永远可点击，进入对应模块默认视图。提醒区根据当前数据展示数量或状态，有待处理内容时点击直达结果，无待处理内容时不跳转，只给轻提示。

## 主入口行为

课程表：
点击整块进入课程表，默认定位到当日所在周，并高亮今日列。主入口中不再放“查看今日课程”子入口，避免和下方今日课程信息提示重复。

费用统计：
点击进入费用统计默认总览/筛选头，用于主动核账。欠缴、结课余额等具体异常不依赖用户手动筛选，而由下方提醒卡直达。

题库：
点击主块进入题库常用入口。块内可放试题库、导入、组卷等子入口；子入口必须直接跳转到对应页面。

云同步：
点击进入同步控制台默认视图。同步异常由下方云同步提醒卡直达异常项。

## 今日提醒行为

今日课程信息提示：
默认展示首位老师的今日课程。每节课程行显示时间段、地点/机构、老师、课程名称。点击某节课后进入课程表，定位到该日期所在周，并滚动或高亮到该课程框。

学生费用欠缴：
展示欠缴学生数量。数量大于 0 时点击进入费用统计，并直接展示“欠缴学生结果集”，不需要用户再手动筛选。数量为 0 时不跳转，提示“暂无欠缴学生”。

结课学生学费剩余：
展示已结课但仍有学费余额的学生数量。数量大于 0 时点击进入费用统计，并直接展示对应学生明细。数量为 0 时不跳转，提示“暂无结课余额异常”。

题库问题试题编辑：
展示导入后被标记为有问题、且尚未编辑确认的问题试题数量。数量大于 0 时点击进入题库问题队列，优先展示待编辑确认题。数量为 0 时不跳转，提示“暂无问题试题”。

云同步情况：
展示同步状态、待同步/失败数量、最近同步时间。异常或待处理数量大于 0 时点击进入云同步页，并直接展示失败或待处理同步项。正常且无待处理时点击不跳转，提示“当前同步正常”。

## 上下文跳转

今日工作台不能只切换页面，还要把目标上下文传给目标页面。建议通过统一的导航事件 detail 扩展实现：

```ts
window.dispatchEvent(new CustomEvent('navigate-page', {
  detail: {
    page: 'revenue-statistics',
    context: { mode: 'arrears' }
  }
}));
```

为兼容现有调用，`navigate-page` 仍支持直接传 page key 字符串。新的对象形式用于携带上下文。

目标页面根据上下文进入指定状态：

- `course-calendar`: `date`, `scheduleId`, `highlightToday`
- `revenue-statistics`: `mode: arrears | closed-balance`
- `question-bank-preview`: `mode: problem-questions`
- `cloud-sync`: `mode: issues | pending`

## 数据来源

今日课程：
优先读取排课明细中的当日课程，默认按首位老师分组展示。需要保留点击单节课程所需的 `scheduleId`、日期和时间。

学生费用欠缴：
从学生缴费、课程/排课费用明细和费用统计底层数据计算。结果集至少包含学生、欠缴金额、关联课程/排课、最近上课日期，方便进入后核对。

结课学生学费剩余：
基于课程状态“已结课”和学生余额/课程费用结算结果计算。结果集至少包含学生、课程、剩余金额、最后上课日期。

问题试题：
读取题库中导入后标记为问题、且尚未编辑确认或处理关闭的试题。

云同步：
读取现有同步状态、待同步队列、失败队列和最近同步时间。

## 空状态与反馈

提醒卡数量为 0 或状态正常时，点击不跳转，只提示当前没有待处理项。提示应短而明确，不弹复杂确认框。

主入口不使用空状态阻断，始终可以进入对应模块。

## 测试

需要覆盖以下场景：

- 左侧导航中“今日工作台”为一级入口，一次点击进入。
- 今日工作台四个主入口能进入对应页面。
- 课程表主入口定位当日所在周并高亮今日列。
- 今日课程行点击后进入课程表并定位到对应课程。
- 欠缴数量大于 0 时进入费用统计并直接显示欠缴结果集。
- 欠缴数量为 0 时不跳转并提示暂无欠缴学生。
- 结课余额数量大于 0 时进入费用统计并直接显示对应明细。
- 结课余额数量为 0 时不跳转并提示暂无结课余额异常。
- 问题试题数量大于 0 时进入题库问题队列。
- 问题试题数量为 0 时不跳转并提示暂无问题试题。
- 云同步异常时进入同步异常列表；正常时不跳转并提示当前同步正常。
